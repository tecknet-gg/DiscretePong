## VGA Driver

I'm using a 640x480 @60Hz spec for my VGA screen, probably the most common. Its driven by a 25.175MHz osscillator. For the cooridinate system we have two 10 bit counters (3x4bit counters ripple chained for each axis):


<img width="560" height="795" alt="image" src="https://github.com/user-attachments/assets/7cc32e43-8a5c-4a55-918b-fd0d7b833c74" />

The master clock feeds into the Horizontal counters, and the the Horizontal clear signal acts as the clock for the Vertical counters, so the vertical only increments after each row. 

The clear signal is generated by some simple AND gates for bit decoding. When a certain bit pattern is noticed, it pulls low and resets the counters (active low). Since the HSYNC and VSYNC generation logic uses NANDs now, I might eventually switch out the ANDs for NANDs in the spirit of cost optimising.


<img width="852" height="619" alt="image" src="https://github.com/user-attachments/assets/e3fe672a-b701-4792-85a8-7dd3f4ac11e5" />

For the HSYNC and VSYNC generation I'm using some NANDs to do the same as with the clear, to decode the necessrary number in bits (was originally using a 10 bit comparator (3x4bit) for each side of the inequality (you need to pull the signal low between a certain threshold). The propagation delay was too high, so I switched to the NAND gate implementation, that then sets and resets a simple S-R Latch (using a D-Type Flip Flop). Produces a reliable HSYNC and VSYNC signal.
